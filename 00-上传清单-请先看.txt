【怎么上传】
  1. 用 Wallace-chan 账号登录 GitHub，打开你的仓库
  2. Add file → Upload files
  3. 把本文件夹「GitHub上传包-沙田三程序」里的**全部内容**拖进去
     （所有 .py、requirements.txt、data 文件夹、.github 文件夹）
  4. Mac 须按 Cmd + Shift + . 才能看到隐藏的 .github 文件夹
  5. Commit message 可填：Add Shatin weather three post programs
  6. Commit changes

  ⚠️ 仓库根目录结构应是平的，例如：
     你的仓库/generate_social_posts.py
     你的仓库/shatin_weather.py
     你的仓库/.github/workflows/social-posts.yml
     不要多包一层「GitHub上传包-沙田三程序」文件夹再上传。

══════════════════════════════════════════════════════════════
一、共用模块（三个程序都依赖，只上传一份）
══════════════════════════════════════════════════════════════
  □ shatin_weather.py      沙田自动气象站数据（HKO 开放数据）
  □ hko_overview.py        天文台全港概况（警告、预报）
  □ deepseek_utils.py      DeepSeek API（可选；无 Key 用本地模板）
  □ shatin_events.py       沙田官方活动（文化博物馆 / 大会堂 / 节目表）
  □ requirements.txt       Python 依赖
  □ .gitignore             忽略 output/、.env 等

══════════════════════════════════════════════════════════════
二、程序 1 · 社交帖文 + 配图（小红书 / IG / FB 同一篇）
══════════════════════════════════════════════════════════════
  主程序：
  □ generate_social_posts.py

  配图相关：
  □ image_utils.py
  □ social_image_styles.py
  □ poster_compose.py

  状态文件：
  □ data/social_state.json

  本地测试：
    python generate_social_posts.py
    输出：output/post_*_social.txt、output/images/*.png

  GitHub Actions：
  □ .github/workflows/social-posts.yml
    定时：每天 HKT 08:00、12:00、18:00
    产物：Artifacts → shatin-social-*

══════════════════════════════════════════════════════════════

══════════════════════════════════════════════════════════════

══════════════════════════════════════════════════════════════
五、GitHub Actions 工作流（共 3 个，都在 .github/workflows/）
══════════════════════════════════════════════════════════════
  □ social-posts.yml       → 程序 1（含配图，最耗时）
  □ cantonese-posts.yml    → 程序 2
  □ multilang-posts.yml    → 程序 3

  共用设置：
  □ Settings → Secrets → DEEPSEEK_API_KEY（可选，不设则用免费模板）

  手动 Run：
  · 三个 workflow 都支持 Run workflow 即时测试
  · 手动跑**不会**把 state 写回仓库，不影响定时三次排程
